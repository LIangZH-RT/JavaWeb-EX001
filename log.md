## 2026-05-12 用户信息管理补充

- 完成用户信息注册、查询、更新、注销的闭环：
  - `/register` 保持独立注册入口，新增账号默认写入 `ACTIVE` 状态。
  - `/UserSearch` 登录后展示本人信息；普通用户只能查询本人信息。
  - `admin` 访问 `/UserSearch` 时可以查看全部用户列表。
  - 用户信息更新只允许更新本人信息，可修改用户名、性别、邮箱，并可选择留空密码表示不修改密码。
  - 用户删除改为注销处理，不物理删除数据库记录，注销后状态为 `CANCELLED`。

- 数据库与仓储层调整：
  - `Users` 表新增 `status varchar(20) not null default 'ACTIVE'` 字段。
  - 启动初始化时会自动给旧表补 `status` 字段。
  - `UserRepository` 新增按 ID 查询、更新用户信息、注销用户方法。
  - `SqlConnector` 增加对应门面方法，业务层继续通过统一入口访问数据库。

- 权限与安全规则：
  - `LoginFilter` 追加拦截 `/UserSearch`，未登录不能查看或修改用户信息。
  - `LoginFilter` 会校验 session 中用户仍为正常状态，已注销账号的旧 session 会被清理。
  - `AdminFilter` 校验管理员账号仍为正常状态；普通用户误进管理员功能只提示无权限，不强制退出。
  - 用户信息更新和注销表单都使用已有 CSRF token 校验。
  - 普通用户不能把用户名修改为 `admin`，避免通过用户名规则越权。
  - 已注销账号不能再次登录；注册时遇到已注销同名账号会提示更换用户名。

- 页面调整：
  - 重写 `jsp/UserSearch.jsp`，分为“我的信息”和管理员“全部用户”区域。
  - 普通用户页面提供“修改信息”和“注销账号”。
  - 管理员列表展示用户状态，并提供对正常账号的注销操作；其他用户信息更新仍限制为本人。
  - `app.css` 增加个人信息表格和弱提示文本样式。

- 验证结果：
  - 已执行 `mvn test`，构建成功，当前项目无测试用例。
